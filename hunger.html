Hunger System

Hunger

In ExPetrum the hunger mechanics are different from vanilla. The player has 2000 calories which slowly go down as time progresses.
The calories value has no maximum and could in theory go above 2000.
The calories are removed by the following formula:
  • The base removal value is 0.05 per tick
  • If the player is sprinting this value is multiplied by 2
  • If time was skipped(for example the player slept) the removed value is calculated as ticksSkipped * value / 2
  • If a player places a block the calories are reduced by 0.1
  • If a player breaks a block the calories are reduced by 1
If the calories are less than or equal to 0 the player takes 1 point of Starvation damage every 2 seconds.
The hunger is displayed in the bottom-left of the screen and the text roughly represents the percentage of calories the player has left:
  • Saturated: Infinity - 100%
  • Not Hungry: 100% - 83%
  • Peckish: 83% - 66%
  • Slightly Hungry: 66% - 50%
  • Hungry: 50% - 33%
  • Very Hungry: 33% - 16%
  • Starving: 16% - 0%

Food

Each food has a certain amount of calories associated with it's weight. Every time a player eats they only eat enough to fill them to 2000.
